# Skigen v1.0.0 — Requirements Specification

**Status:** Draft  
**Date:** 2026-05-01  
**Standard:** EES (Energy-Efficient Software)

---

## 1. Project Identity

**Name:** Skigen  
**Language:** Latest C++ (currently C++23)  
**Architecture:** Header-only  
**Sole Dependency:** Eigen (numerical backend)  
**License:** MIT  

**Value Proposition:** An energy-efficient machine learning library built on Eigen that delivers a high-performance, native framework to maximize computational density through resource-lean, agent-ready AI.

**Core Objective:** Mirror the scikit-learn API with absolute mathematical precision and zero interpreter tax.

---

## 2. Non-Negotiable Technical Guardrails

### 2.1 EES Template Policy

| Rule | Requirement |
|------|-------------|
| **Fully Templatized** | All classes accept a `Scalar` type parameter (default: `double`). Switching to `float` yields 2× SIMD density. |
| **Zero-Copy Inputs** | Use `Eigen::Ref<const MatrixType>` for all read-only inputs — supports memory mapping and sub-blocks without copying. |
| **Expression Templates** | No raw loops over matrix elements; rely on Eigen's internal vectorization. |
| **Memory Discipline** | Minimize heap allocations; favor in-place operations; provide `_inplace` variants where applicable. |
| **No Compiled Sources** | Every component is a header file. No `.cpp` files in `Skigen/`. |

### 2.2 Mathematical Parity

Results must be **bit-level consistent** with scikit-learn for identical inputs and default parameters. Verification is done via Python parity tests that compare skigen output against scikit-learn output on reference datasets.

### 2.3 Compiler Requirements

- C++23 or later.
- Must compile cleanly (zero warnings) with `-Wall -Wextra -Wpedantic` on GCC ≥ 13, Clang ≥ 17, and MSVC ≥ 19.38.

## 4. Core Architecture

### 4.1 Static Polymorphism via CRTP

All estimators use the Curiously Recurring Template Pattern for zero-cost abstractions. No virtual functions, no vtable overhead.

**Base Class Hierarchy:**

```
Estimator<Derived, Scalar>         — fit(), get_params()
├── Transformer<Derived, Scalar>   — transform(), fit_transform(), inverse_transform()
└── Predictor<Derived, Scalar>     — predict(), score()
```

### 4.2 Type Aliases (src/Core/Traits.h)

Every class must define these canonical types:

```cpp
using ScalarType = Scalar;
using MatrixType = Eigen::Matrix<Scalar, Eigen::Dynamic, Eigen::Dynamic>;
using VectorType = Eigen::Matrix<Scalar, Eigen::Dynamic, 1>;
using RowVectorType = Eigen::Matrix<Scalar, 1, Eigen::Dynamic>;
using IndexType = Eigen::Index;
```

### 4.3 Concepts (src/Core/Concepts.h)

C++23 concepts to constrain template arguments:

```cpp
template <typename T>
concept EigenScalar = std::is_floating_point_v<T>;

template <typename T>
concept TransformerLike = requires(T t, const typename T::MatrixType& X) {
    { t.fit(X) } -> std::same_as<T&>;
    { t.transform(X) } -> std::same_as<typename T::MatrixType>;
};

template <typename T>
concept PredictorLike = requires(T t, const typename T::MatrixType& X) {
    { t.fit(X, std::declval<const typename T::VectorType&>()) } -> std::same_as<T&>;
    { t.predict(X) } -> std::same_as<typename T::VectorType>;
};
```

### 4.4 Base Class: Estimator (src/Core/Base.h)

```cpp
namespace Skigen {

template <typename Derived, typename Scalar = double>
class Estimator {
public:
    using ScalarType = Scalar;
    using MatrixType = Eigen::Matrix<Scalar, Eigen::Dynamic, Eigen::Dynamic>;
    using VectorType = Eigen::Matrix<Scalar, Eigen::Dynamic, 1>;
    using RowVectorType = Eigen::Matrix<Scalar, 1, Eigen::Dynamic>;
    using IndexType = Eigen::Index;

    bool is_fitted() const noexcept { return fitted_; }

protected:
    bool fitted_ = false;
    IndexType n_features_in_ = 0;

    void check_is_fitted() const {
        if (!fitted_) throw std::runtime_error("Estimator not fitted");
    }

    void validate_feature_count(const Eigen::Ref<const MatrixType>& X) const {
        if (X.cols() != n_features_in_) {
            throw std::invalid_argument("Feature count mismatch");
        }
    }
};

} // namespace Skigen
```

### 4.5 Base Class: Transformer (src/Core/Base.h)

```cpp
namespace Skigen {

template <typename Derived, typename Scalar = double>
class Transformer : public Estimator<Derived, Scalar> {
public:
    using typename Estimator<Derived, Scalar>::MatrixType;

    auto& fit(const Eigen::Ref<const MatrixType>& X) {
        return static_cast<Derived*>(this)->fit_impl(X);
    }

    MatrixType transform(const Eigen::Ref<const MatrixType>& X) const {
        return static_cast<const Derived*>(this)->transform_impl(X);
    }

    MatrixType fit_transform(const Eigen::Ref<const MatrixType>& X) {
        fit(X);
        return transform(X);
    }

    MatrixType inverse_transform(const Eigen::Ref<const MatrixType>& X) const {
        return static_cast<const Derived*>(this)->inverse_transform_impl(X);
    }
};

} // namespace Skigen
```

### 4.6 Base Class: Predictor (src/Core/Base.h)

```cpp
namespace Skigen {

template <typename Derived, typename Scalar = double>
class Predictor : public Estimator<Derived, Scalar> {
public:
    using typename Estimator<Derived, Scalar>::MatrixType;
    using typename Estimator<Derived, Scalar>::VectorType;

    auto& fit(const Eigen::Ref<const MatrixType>& X,
              const Eigen::Ref<const VectorType>& y) {
        return static_cast<Derived*>(this)->fit_impl(X, y);
    }

    VectorType predict(const Eigen::Ref<const MatrixType>& X) const {
        return static_cast<const Derived*>(this)->predict_impl(X);
    }

    Scalar score(const Eigen::Ref<const MatrixType>& X,
                 const Eigen::Ref<const VectorType>& y) const {
        return static_cast<const Derived*>(this)->score_impl(X, y);
    }
};

} // namespace Skigen
```

## 5. v1.0.0 Component Specifications

### 5.1 preprocessing::StandardScaler

**scikit-learn reference:** `sklearn.preprocessing.StandardScaler`

#### Constructor Parameters

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `with_mean` | `bool` | `true` | Center data before scaling. |
| `with_std` | `bool` | `true` | Scale data to unit variance. |

> Note: scikit-learn's `copy` parameter is handled by the zero-copy `Eigen::Ref` pattern and the explicit `transform_inplace` variant.

#### Fitted Attributes

| Attribute | Type | Description |
|-----------|------|-------------|
| `mean_` | `RowVectorType` | Per-feature mean. Computed when `with_mean=true` or `with_std=true`. |
| `var_` | `RowVectorType` | Per-feature variance (biased, ddof=0). Computed when `with_std=true`. |
| `scale_` | `RowVectorType` | Per-feature scale factor = `sqrt(var_)`. Computed when `with_std=true`. |
| `n_samples_seen_` | `IndexType` | Number of samples processed during fit. |
| `n_features_in_` | `IndexType` | Number of features (inherited from Estimator). |

#### Formulas

**Fit:**
$$\mu_j = \frac{1}{n} \sum_{i=1}^{n} X_{ij}$$

$$\sigma_j^2 = \frac{1}{n} \sum_{i=1}^{n} (X_{ij} - \mu_j)^2$$

$$s_j = \sqrt{\sigma_j^2}$$

**Transform:**
$$Z_{ij} = \frac{X_{ij} - \mu_j}{s_j}$$

Where $\mu_j = 0$ if `with_mean=false`, and $s_j = 1$ if `with_std=false`.

**Inverse Transform:**
$$X_{ij} = Z_{ij} \cdot s_j + \mu_j$$

#### Near-Zero Variance Handling

A feature's scale is considered degenerate when:

$$s_j < 10 \cdot \varepsilon$$

where $\varepsilon$ is `std::numeric_limits<Scalar>::epsilon()`. In this case, $s_j$ is set to $1.0$ to prevent division by zero or near-zero values.

This matches scikit-learn's `_handle_zeros_in_scale()` behavior.

#### Required Methods

```cpp
template <typename Scalar = double>
class StandardScaler : public Transformer<StandardScaler<Scalar>, Scalar> {
public:
    // Configuration
    StandardScaler(bool with_mean = true, bool with_std = true);

    // Core API (called by base class)
    auto& fit_impl(const Eigen::Ref<const MatrixType>& X);
    MatrixType transform_impl(const Eigen::Ref<const MatrixType>& X) const;
    MatrixType inverse_transform_impl(const Eigen::Ref<const MatrixType>& X) const;

    // EES extension: in-place transform
    void transform_inplace(Eigen::Ref<MatrixType> X) const;
    void inverse_transform_inplace(Eigen::Ref<MatrixType> X) const;

    // Accessors
    const RowVectorType& mean() const;
    const RowVectorType& var() const;
    const RowVectorType& scale() const;
    IndexType n_samples_seen() const;
};
```

#### Implementation Constraints

- `fit_impl` must use Eigen's `.colwise().mean()` and column-wise variance via expression templates.
- `transform_impl` must use Eigen broadcasting: `(X.rowwise() - mean_).array().rowwise() / scale_.array()`.
- No raw loops.
- `transform_inplace` must modify `X` directly using `Eigen::Ref<MatrixType>`.

### 5.2 linear_model::LinearRegression

**scikit-learn reference:** `sklearn.linear_model.LinearRegression`

#### Constructor Parameters

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `fit_intercept` | `bool` | `true` | Whether to calculate the intercept. |

> Note: scikit-learn's `copy_X`, `n_jobs`, `positive` are omitted for v1.0.0. `copy_X` is handled by zero-copy semantics. `positive` (NNLS) and `n_jobs` are deferred to v1.1.

#### Fitted Attributes

| Attribute | Type | Description |
|-----------|------|-------------|
| `coef_` | `VectorType` | Regression coefficients, shape `(n_features,)`. |
| `intercept_` | `Scalar` | Intercept term. Set to `0` if `fit_intercept=false`. |
| `rank_` | `IndexType` | Effective rank of the input matrix. |
| `singular_values_` | `VectorType` | Singular values of X. |
| `n_features_in_` | `IndexType` | Number of features (inherited). |

#### Formulas

**Ordinary Least Squares via SVD:**

$$\hat{w} = \arg\min_w \|Xw - y\|_2^2$$

Solved using Eigen's `JacobiSVD` or `BDCSVD` decomposition:

$$X = U \Sigma V^T$$
$$\hat{w} = V \Sigma^{-1} U^T y$$

**Intercept calculation** (when `fit_intercept=true`):

$$b = \bar{y} - \bar{X} \cdot \hat{w}$$

Where $\bar{y}$ and $\bar{X}$ are the means of the target and features respectively. The data is centered before SVD to compute the intercept.

**Prediction:**
$$\hat{y} = Xw + b$$

**Score (R²):**
$$R^2 = 1 - \frac{\sum_i (y_i - \hat{y}_i)^2}{\sum_i (y_i - \bar{y})^2}$$

#### Required Methods

```cpp
template <typename Scalar = double>
class LinearRegression : public Predictor<LinearRegression<Scalar>, Scalar> {
public:
    LinearRegression(bool fit_intercept = true);

    auto& fit_impl(const Eigen::Ref<const MatrixType>& X,
                   const Eigen::Ref<const VectorType>& y);
    VectorType predict_impl(const Eigen::Ref<const MatrixType>& X) const;
    Scalar score_impl(const Eigen::Ref<const MatrixType>& X,
                      const Eigen::Ref<const VectorType>& y) const;

    // Accessors
    const VectorType& coef() const;
    Scalar intercept() const;
    IndexType rank() const;
    const VectorType& singular_values() const;
};
```

#### Implementation Constraints

- Use `Eigen::BDCSVD` for matrices where `min(m,n) > 16`, `Eigen::JacobiSVD` otherwise.
- Compute `ComputeThinU | ComputeThinV` for memory efficiency.
- Center X and y in-place (on copies) when `fit_intercept=true`.

Standalone function templates in `Skigen::metrics` namespace. No classes.

#### Regression Metrics

} // namespace Skigen::metrics

## 6. Testing Strategy

### 6.1 Parity Tests

For each estimator:
1. A Python script generates reference inputs and outputs using scikit-learn.
2. Reference data is stored as CSV or binary (row-major, matching Eigen's default).
3. A C++ test reads the reference data, runs the skigen estimator, and asserts bit-level parity.

**Tolerance:** For `double`, absolute tolerance of `1e-12`. For `float`, absolute tolerance of `1e-5`.

### 6.2 Unit Tests

Framework: A lightweight header-only test framework or Catch2/doctest.

Required test cases per component:

**StandardScaler:**
- Fit on known data, verify `mean_`, `var_`, `scale_`.
- Transform produces correct Z-scores.
- `inverse_transform(transform(X)) ≈ X` (round-trip).
- `fit_transform(X) == fit(X).transform(X)`.
- Near-zero variance features produce `scale_ = 1.0`.
- `with_mean=false` leaves data uncentered.
- `with_std=false` leaves data unscaled.
- `transform_inplace` matches `transform` output.
- Single-feature input (n_features=1).
- Single-sample input (n_samples=1).
- Feature count mismatch throws.
- Not-fitted estimator throws.

**LinearRegression:**
- Perfect linear data recovers exact coefficients.
- `fit_intercept=false` on centered data.
- Underdetermined system (n_samples < n_features).
- Overdetermined system (n_samples >> n_features).
- `score()` returns correct R².
- `predict()` matches `X @ coef + intercept`.
- Single feature (simple regression).
- Not-fitted estimator throws.

### 6.3 Compile-Time Tests

- `StandardScaler<float>` and `StandardScaler<double>` both compile.
- `LinearRegression<float>` and `LinearRegression<double>` both compile.
- Concept checks validate that estimators satisfy `TransformerLike` / `PredictorLike`.

## 8. Build System

### 8.1 CMake Configuration

- Minimum CMake version: 3.20.
- The library itself requires no build (header-only).
- CMake targets: `Skigen::skigen` (interface library).
- `find_package(Eigen3 REQUIRED)` for the Eigen dependency.
- Options:
  - `SKIGEN_BUILD_TESTS` (default: ON in dev, OFF in install)
  - `SKIGEN_BUILD_BENCHMARKS` (default: OFF)
  - `SKIGEN_BUILD_EXAMPLES` (default: OFF)

### 8.2 Installation

Standard CMake install:
```cmake
install(DIRECTORY skigen DESTINATION ${CMAKE_INSTALL_INCLUDEDIR})
```

Exported target allows:
```cmake
find_package(skigen REQUIRED)
target_link_libraries(my_app PRIVATE Skigen::skigen)
```
