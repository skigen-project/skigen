# Skigen v1.2.0 — Requirements

Running list of work deferred past v1.1.0. Not a committed scope yet.

## Documentation

### Rendered example plots in the guide pages (carried from v1.1.0)

Include the SkigenPlot-rendered example figures in the documentation site.

**Done (the core feature is in place):**
- `examples/CMakeLists.txt` exposes `skigen_enable_plot(<target> <stem>)` and the
  `skigen_render_example_plots` target that runs each registered plot example to
  emit `<stem>_dark.png` / `<stem>_light.png` into
  `doc/website/static/img/examples/` (see that directory's `README.md`).
- Three examples are registered and their figures embedded via `ThemedImage`:
  `kmeans` → `guide/kmeans.mdx`, `tsne` → `guide/tsne.mdx`,
  `pca_clustering` → `guide/pca.mdx`.
- **The PNGs are git-ignored, not committed.** Both `staging.yml` and `main.yml`
  run `skigen_render_example_plots` (mne-cpp pre-built Qt 6.11.1 under `xvfb`)
  right before `npm run build`, so the published site gets fresh figures.
- Local rendering needs **Qt 6.11.1** (`-DCMAKE_PREFIX_PATH=~/Qt/6.11.1/macos`;
  the local 6.11.0 install is incomplete — missing `QtWidgets.framework`).

**Remaining:**
- Grow the gallery: register more examples with `skigen_enable_plot(<tgt> <stem>)`
  and reference each from its guide page (e.g. SVM decision regions, more
  manifold embeddings). Add the `ThemedImage` reference and the CI render
  registration together so the published site never links a missing image.
