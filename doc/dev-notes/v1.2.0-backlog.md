# Skigen v1.2.0 — Backlog

Running list of work deferred past v1.1.0. Not a committed scope yet.

## Documentation

### Rendered example plots in the guide pages (carried from v1.1.0)

Include the SkigenPlot-rendered example figures in the documentation site.

**Status / what already exists:**
- `examples/CMakeLists.txt` exposes `skigen_enable_plot(<target> <stem>)` and a
  `skigen_render_example_plots` target that runs each registered plot example to
  emit `<stem>_dark.png` / `<stem>_light.png` into
  `doc/website/static/img/examples/` (see that directory's `README.md` for the
  exact commands and the `ThemedImage` snippet to reference them).
- Currently only `ex_kmeans` (`kmeans` stem) is registered.

**Remaining for the next release:**
- Render the PNGs on a Qt-enabled machine / in CI (local dev boxes may lack the
  Qt 6 runtime) and commit them under `doc/website/static/img/examples/`.
- Reference the rendered figures from the corresponding guide pages with
  `ThemedImage` (start with `guide/kmeans.mdx`). Only add the reference once the
  PNGs are committed so the published site never links a missing image.
- Add `skigen_enable_plot(<target> <stem>)` registrations to more examples so
  the gallery grows (decomposition/PCA, manifold embeddings, clustering, etc.).
- Optional: wire `skigen_render_example_plots` into the docs/release CI so the
  images regenerate automatically.
