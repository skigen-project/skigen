# Skigen v1.2.0 — Backlog

Running list of work deferred past v1.1.0. Not a committed scope yet.

## Documentation

### Rendered example plots in the guide pages (carried from v1.1.0)

Include the SkigenPlot-rendered example figures in the documentation site.

**Status / what already exists:**
- `examples/CMakeLists.txt` exposes `skigen_enable_plot(<target> <stem>)` and a
  `skigen_render_example_plots` target that runs each registered plot example to
  emit `<stem>_dark.png` / `<stem>_light.png` into
  `doc/website/static/img/examples/` (see that directory's `README.md` for the
  exact commands and the `ThemedImage` snippet to reference them).
- **`kmeans` is done**: rendered against **Qt 6.11.1** (configure with
  `-DCMAKE_PREFIX_PATH=~/Qt/6.11.1/macos`; the older 6.11.0 install is
  incomplete and fails to load `QtWidgets.framework`), the PNGs are committed,
  and `guide/kmeans.mdx` references them via `ThemedImage`. The Docusaurus site
  builds clean with the figure.

**Remaining:**
- Add `skigen_enable_plot(<target> <stem>)` registrations to more examples so
  the gallery grows (decomposition/PCA, manifold embeddings, etc.), render, and
  reference them from their guide pages — only commit a reference once its PNGs
  exist so the published site never links a missing image.
- Optional: wire `skigen_render_example_plots` into the docs/release CI (using
  the mne-cpp pre-built Qt 6.11.1) so the images regenerate automatically.
