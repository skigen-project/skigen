# Skigen v1.1.0 — Requirements Specification

**Status:** Draft
**Date:** 2026-05-09
**Predecessor:** [v1.0.0-requirements.md](v1.0.0-requirements.md)
**Standard:** EES (Energy-Efficient Software)

---

## 1. Release Theme

**EES focus:** Mixed — *streaming + sparsity + algorithmic breadth + manifold/probabilistic depth*.

v1.1.0 closes the remaining sklearn coverage gaps (Ensembles incl. HistGradientBoosting, full SVM with kernels, MLP, Naive Bayes, Feature Selection, hyperparameter search, Bayesian linear models, manifold learning, calibration) while making one decisive stride on the infrastructure story that defines Skigen's EES identity: **incremental learning** (`partial_fit`) and **sparse matrix support** (`Eigen::SparseMatrix`) extended across clustering, decomposition, and trees.

It also formalises the parity-test harness so that bit-level scikit-learn parity becomes a CI invariant rather than an aspiration, and establishes a one-example-per-estimator rule that drives the documentation site.

### Versioning Policy (this release only)

Skigen has no production users yet. v1.1.0 is therefore explicitly permitted to introduce **breaking changes** where they improve API quality, sklearn parity, or EES infrastructure. The strict-additive semver policy resumes from v1.2.0 onward.

The breaking-change budget is intentional but bounded: each break must be motivated by one of (a) sklearn parity correction, (b) sparse/dense unification, (c) `partial_fit` integration, or (d) elimination of dead default arguments. Cosmetic renames are not in scope.

## 2. Inherited Guardrails

All §2 (Technical Guardrails), §3 (Repository Structure), and §4 (Core Architecture) rules from v1.0.0 remain in force:

- Header-only, Eigen-only dependency, C++23.
- CRTP base hierarchy (`Estimator` → `Transformer` / `Predictor` / `ClusterMixin` / …).
- `Eigen::Ref<const MatrixType>` zero-copy inputs.
- Bit-level scikit-learn parity for default parameters.
- Zero warnings under `-Wall -Wextra -Wpedantic` on GCC ≥ 13, Clang ≥ 17, MSVC ≥ 19.38.

v1.1.0 extensions to the guardrails are listed in §3 below.

## 3. New Cross-Cutting Requirements

### 3.0 Strict scikit-learn Parity (release-blocking)

Skigen v1.1.0 elevates sklearn parity from "best effort" to a **release-blocking contract**. Every estimator, transformer, metric, and utility added in v1.1.0 — and every existing v1.0.0 component touched by §5 refactors — must satisfy *all* of the following:

**Parameter parity.**
- Constructor parameter **names**, **order**, **types**, and **defaults** match the corresponding sklearn class exactly (translated to idiomatic C++: `int` / `Scalar` / `bool` / `std::optional<T>` / dedicated `enum class` for string enums / `std::variant` where sklearn accepts heterogeneous values).
- Every sklearn parameter is represented. Parameters that do not apply to a header-only Eigen world (`copy`, `copy_X`, `precompute_gram` for some solvers) are still accepted and documented as no-ops with a `[[deprecated("no-op in Skigen — zero-copy by construction")]]` annotation, never silently dropped.
- Default values are byte-identical to the sklearn version pinned for parity (see *Reference version* below). When sklearn changes a default in a future release, Skigen tracks the new default in the next minor version and notes the change in `CHANGELOG.md`.
- String-valued parameters become `enum class`; the documented spelling (`"l2"`, `"hinge"`, `"sqrt"`) is the canonical guide-page name.

**Attribute parity.**
- Every sklearn fitted attribute (trailing-underscore name) is exposed as a member accessor with the same shape and dtype semantics. Names use the same `name_` convention.
- Where sklearn returns a 1-D array vs 2-D array depending on input shape, Skigen mirrors the same conditional shape.

**Behavioural parity.**
- Numerical results match sklearn within tolerance (`1e-12` for `double`, `1e-5` for `float`) on the parity test fixtures — for **default parameters** *and* for at least one non-default configuration per parameter that admits a discrete or scalar choice.
- Random-state semantics match: identical `random_state` seeds produce identical splits, initialisations, and shuffles relative to sklearn's documented algorithm (modulo FP non-determinism in genuinely parallel reductions, which is documented per estimator).
- Error conditions match: the same invalid input that raises `ValueError` in sklearn raises `std::invalid_argument` (or a typed Skigen exception) in Skigen, and the message references the same offending parameter.

**Reference version.**
- v1.1.0 pins parity against **scikit-learn 1.7.x** (current stable as of release). The pinned version is recorded in `tests/parity/sklearn_version.txt` and enforced by the regenerator.
- Parity tests fail loudly if executed against a different sklearn minor version during regeneration.

**Coverage gate.** No estimator may close its phase ticket without (a) parity tests for every public method, (b) a unit test exercising every constructor parameter and every fitted attribute, and (c) an example in `examples/<module>/<estimator>.cpp` that runs end-to-end and is referenced from a guide page.

### 3.1 Incremental-Learning Concept

A new C++23 concept extends the estimator taxonomy:

```cpp
template <typename T>
concept IncrementalLike = TransformerLike<T> && requires(T t, const typename T::MatrixType& X) {
    { t.partial_fit(X) } -> std::same_as<T&>;
};
```

A `partial_fit(X)` call must:

- Update internal sufficient statistics in **O(n_samples · n_features)** time and **O(1)** allocations beyond the persistent state.
- Be invariant under batching: the result of `partial_fit(X1).partial_fit(X2)` must be bit-equivalent (within FP tolerance) to `partial_fit([X1; X2])`, matching sklearn's contract.
- Require `n_features_in_` to match the value seen on the first call; throw `std::invalid_argument` otherwise.
- Use Welford / Chan online updates for mean and variance (no naive recomputation, no catastrophic cancellation).

### 3.2 SparseMatrix Support

Estimators that admit a sparse implementation gain a templated overload set:

```cpp
template <typename Scalar = double>
class TruncatedSVD : public Transformer<…> {
    // Existing dense API
    auto& fit_impl(const Eigen::Ref<const MatrixType>& X);

    // New sparse API — same semantics, no copy through dense
    template <int Options, typename StorageIndex>
    auto& fit_impl(const Eigen::SparseMatrix<Scalar, Options, StorageIndex>& X);
};
```

**In-scope sparse-aware estimators (v1.1.0):**

- `Normalizer` — row-wise norm scaling.
- `TruncatedSVD` — already algorithm-of-choice for sparse data in sklearn.
- `PCA` — sparse path falls back to `TruncatedSVD`-style mean-centred ARPACK / randomised solver (sklearn's `arpack` and `randomized` SVD solvers).
- `LinearRegression`, `Ridge`, `Lasso`, `ElasticNet`, `BayesianRidge`, `ARDRegression` — solvers branch to sparse-friendly paths (CG / coordinate descent operating on sparse views).
- `KMeans`, `MiniBatchKMeans` — sparse-aware distance computations using `Eigen::SparseMatrix` row views; centroid updates retain dense storage.
- `DecisionTreeClassifier`, `DecisionTreeRegressor`, `RandomForest*`, `GradientBoosting*` — sparse-aware split finding via column iteration over `SparseMatrix<Scalar, Eigen::ColMajor>`.
- `MultinomialNB`, `BernoulliNB` — text-classification flagship use case.
- `LinearSVC`, `SVC` (kernel SVM with linear/RBF/poly/sigmoid kernels) — operate directly on `SparseMatrix<Scalar, Eigen::RowMajor>`.
- `feature_selection::VarianceThreshold`, `SelectKBest` (with `chi2`) — column-wise statistics over sparse columns.

Out of scope for v1.1.0: sparse `MLP` (sklearn doesn't support either), sparse manifold learners.

**Parity invariant:** Sparse and dense fit paths must produce identical fitted attributes for any input that round-trips through `.toDense()` (within `1e-12` for `double`, `1e-5` for `float`).

### 3.3 Multi-Target Regression

Where sklearn supports it, regressors gain a `MatrixType` overload:

```cpp
auto& fit(const Eigen::Ref<const MatrixType>& X, const Eigen::Ref<const MatrixType>& Y);
MatrixType predict(const Eigen::Ref<const MatrixType>& X) const;
```

Affected v1.0.0 estimators (breaking-change opportunity):

- `LinearRegression`, `Ridge`, `Lasso`, `ElasticNet`, `KNeighborsRegressor`, `DecisionTreeRegressor`.

The single-target API is preserved as a thin overload that wraps `Y` as an `n×1` matrix. Existing call sites that consumed `coef_` as `VectorType` will need to migrate to `MatrixType` (this is the largest documented break in v1.1.0).

### 3.4 Parity Harness

Promote the existing Python reference generators into a first-class CTest target:

- `tests/parity/generate_<estimator>_reference.py` for every v1.0.0 + v1.1.0 estimator.
- Generators write `(input.npy, params.json, expected.npy)` triples to `tests/parity/data/`.
- A CMake target `skigen_parity_tests` reads the triples, runs the C++ estimator with the recorded parameters, and asserts element-wise equality within tolerance.
- The reference data is checked into the repo (small, deterministic seeds) — CI does **not** invoke Python at test time.
- Regenerating data is an explicit `cmake --build build --target skigen_regenerate_parity` step, gated on `find_package(Python3)` and `pip show scikit-learn`.

**Coverage gate:** every estimator in v1.1.0 must ship with at least one parity test before its phase ticket can be closed.

## 4. New Component Specifications

> Each component below follows the v1.0.0 contract: Scalar-templated, CRTP base, zero-copy `Eigen::Ref` inputs, no virtuals, sklearn-compatible defaults.

### 4.1 ensemble::RandomForest

**sklearn reference:** `RandomForestClassifier`, `RandomForestRegressor`.

Built on the existing `DecisionTreeClassifier` / `DecisionTreeRegressor`. Each tree owns its bootstrap sample and feature subspace; predictions average (regressor) or majority-vote / soft-vote (classifier).

| Parameter | Type | Default | Notes |
|---|---|---|---|
| `n_estimators` | `int` | `100` | sklearn 1.x default. |
| `criterion` | enum | `Gini` / `MSE` | Inherits tree criteria. |
| `max_depth` | `std::optional<int>` | `nullopt` | Pass-through. |
| `min_samples_split` | `int` | `2` | |
| `min_samples_leaf` | `int` | `1` | |
| `max_features` | `MaxFeatures` | `Sqrt` (clf) / `OneThird` (reg) | enum: `All`, `Sqrt`, `Log2`, `Fraction(double)`, `Count(int)`. |
| `bootstrap` | `bool` | `true` | |
| `oob_score` | `bool` | `false` | When `true`, exposes `oob_score_`. |
| `random_state` | `std::optional<uint64_t>` | `nullopt` | |
| `n_jobs` | `int` | `1` | Thread-pool dispatch via `std::async`; trees are embarrassingly parallel. |

**Implementation constraints:**

- Tree fits run on independent sub-samples; no shared mutable state during fit.
- `feature_importances_` is the mean Gini/MSE decrease across trees.
- Memory budget per tree: amortised O(n_samples) for the bootstrap indices vector — no per-tree copy of `X`.

### 4.2 ensemble::GradientBoosting + HistGradientBoosting

**sklearn reference:** `GradientBoostingClassifier`, `GradientBoostingRegressor`, `HistGradientBoostingClassifier`, `HistGradientBoostingRegressor`.

Stage-wise additive model on regression trees (depth-bounded), with line-search shrinkage.

| Parameter | Type | Default |
|---|---|---|
| `loss` | enum | `LogLoss` (clf) / `SquaredError` (reg) |
| `learning_rate` | `Scalar` | `0.1` |
| `n_estimators` | `int` | `100` |
| `subsample` | `Scalar` | `1.0` (stochastic GB when < 1) |
| `max_depth` | `int` | `3` |
| `min_samples_leaf` | `int` | `1` |
| `random_state` | `std::optional<uint64_t>` | `nullopt` |
| `validation_fraction` | `Scalar` | `0.1` |
| `n_iter_no_change` | `std::optional<int>` | `nullopt` (early stopping) |

**Implementation constraints:**

- Reuse `DecisionTreeRegressor` as the base learner regardless of task (classification stages fit pseudo-residuals).
- Maintain a running prediction vector; incremental update per stage in O(n_samples).

**HistGradientBoosting** uses sklearn's histogram-based algorithm: features are binned into `max_bins` (default `255`) buckets up-front, and split finding operates on the histogram of gradients/hessians per bin. This makes training near-linear in n_samples and is required for parity with sklearn's recommended GB path on large datasets.

| HGB-specific parameter | Default |
|---|---|
| `max_bins` | `255` |
| `max_iter` | `100` |
| `max_leaf_nodes` | `31` |
| `min_samples_leaf` | `20` |
| `l2_regularization` | `0.0` |
| `early_stopping` | `"auto"` |
| `categorical_features` | `nullopt` |
| `monotonic_cst` | `nullopt` |

HistGradientBoosting must accept `Eigen::SparseMatrix` directly (sklearn does, since 1.5).

### 4.3 naive_bayes::{Gaussian,Multinomial,Bernoulli}NB

| Estimator | fitted attributes | sparse-aware |
|---|---|---|
| `GaussianNB` | `theta_` (n_classes × n_features), `var_`, `class_prior_` | no |
| `MultinomialNB` | `feature_log_prob_`, `class_log_prior_`, `alpha` smoothing | **yes** |
| `BernoulliNB` | `feature_log_prob_`, `class_log_prior_`, `binarize`, `alpha` | **yes** |

All three support `partial_fit(X, y, classes=...)` for streaming text-classification workflows.

### 4.4 svm::{LinearSVC, LinearSVR, SVC, SVR, NuSVC, NuSVR, OneClassSVM}

**Linear SVMs (liblinear-style coordinate descent).**

| Class | Common parameters |
|---|---|
| `LinearSVC` | `C=1.0`, `loss="squared_hinge"`, `penalty="l2"`, `dual="auto"`, `tol=1e-4`, `max_iter=1000`, `multi_class="ovr"`, `fit_intercept=true`, `intercept_scaling=1.0`, `class_weight=nullopt`, `random_state=nullopt`. |
| `LinearSVR` | `C=1.0`, `loss="epsilon_insensitive"`, `epsilon=0.0`, `tol=1e-4`, `max_iter=1000`, `dual="auto"`, `fit_intercept=true`, `intercept_scaling=1.0`, `random_state=nullopt`. |

**Kernel SVMs (libsvm-style SMO solver).**

A new `SMO` solver in `src/SVM/Detail/SMO.h` implements Sequential Minimal Optimization with shrinking and caching, matching libsvm's algorithm (which is what sklearn ships under the hood).

| Class | Defaults that diverge or matter |
|---|---|
| `SVC` | `C=1.0`, `kernel="rbf"`, `degree=3`, `gamma="scale"`, `coef0=0.0`, `shrinking=true`, `probability=false`, `tol=1e-3`, `cache_size=200` (MB), `class_weight=nullopt`, `decision_function_shape="ovr"`, `break_ties=false`, `random_state=nullopt`. |
| `SVR` | Same kernel/gamma/coef0/degree/shrinking/cache_size as `SVC`; `epsilon=0.1`, `C=1.0`, `tol=1e-3`. |
| `NuSVC` | Replaces `C` with `nu=0.5`. |
| `NuSVR` | Replaces `C` with `nu=0.5`, plus `C=1.0` upper bound on Lagrange multipliers. |
| `OneClassSVM` | `kernel="rbf"`, `nu=0.5`. Fit/predict only, no `score`. |

**Kernel implementations.** `Linear`, `RBF`, `Polynomial`, `Sigmoid`, plus a `Precomputed` mode that takes a Gram matrix directly. Kernels are dispatched at compile time via a `Kernel` policy template — no runtime branching in the SMO inner loop.

**Constraints:**

- Sparse-aware via `Eigen::SparseMatrix` overload for all linear and kernel SVMs.
- `SVC` supports binary + OvO multiclass (matching sklearn) — `decision_function_shape="ovr"` reshapes the OvO output as sklearn does.
- `decision_function(X)` returns `VectorType` for binary, `MatrixType` for multiclass.
- `predict_proba` requires `probability=true` and triggers Platt scaling (calibrated via internal 5-fold CV — same algorithm as libsvm).
- SMO loop uses the second-order working-set selection from libsvm; identical seed produces identical iterate sequence (modulo FP non-associativity).

### 4.5 neural_network::MLP{Classifier,Regressor}

Dense feed-forward network, trained with mini-batch SGD + Adam.

| Parameter | Type | Default |
|---|---|---|
| `hidden_layer_sizes` | `std::vector<int>` | `{100}` |
| `activation` | enum | `ReLU` (also `Tanh`, `Logistic`, `Identity`) |
| `solver` | enum | `Adam` (also `SGD`, `LBFGS`) |
| `alpha` | `Scalar` | `1e-4` (L2 reg) |
| `batch_size` | `int` | `min(200, n_samples)` |
| `learning_rate_init` | `Scalar` | `1e-3` |
| `max_iter` | `int` | `200` |
| `tol` | `Scalar` | `1e-4` |
| `random_state` | `std::optional<uint64_t>` | `nullopt` |
| `early_stopping` | `bool` | `false` |
| `n_iter_no_change` | `int` | `10` |

**Constraints:**

- All weights/biases held as Eigen matrices; forward/backward written entirely as expression-template ops (no raw element loops).
- Adam state vectors (`m_`, `v_`) are persistent across `partial_fit` calls.
- LBFGS solver wraps Eigen's unsupported LBFGS module if available; otherwise gated behind a CMake feature flag.
- Verifiable parity with sklearn's MLP is approximate (FP non-determinism in mini-batch order); parity test uses fixed seed + `solver=lbfgs` for determinism.

### 4.6 feature_selection

Lightweight transformers — pure statistics, no fitting cost beyond a single pass.

| Class | Strategy |
|---|---|
| `VarianceThreshold` | Remove features with variance < `threshold` (default `0.0`). Sparse-aware. |
| `SelectKBest` | Keep top-k features by univariate score. Score functions: `f_classif`, `f_regression`, `chi2`, `mutual_info_classif`, `mutual_info_regression` (latter two stretch). |
| `SelectFromModel` | Wrap an estimator exposing `coef_` or `feature_importances_`; threshold on `mean`, `median`, or numeric. |
| `RFE` | Recursive feature elimination — fit estimator, remove weakest, repeat until `n_features_to_select`. |

### 4.7 linear_model::{BayesianRidge, ARDRegression}

Probabilistic linear regression with automatic relevance determination of regularisation strength.

**`BayesianRidge`** fits a Gaussian linear model with Gamma priors on noise precision (`alpha`) and weight precision (`lambda`). Both hyperpriors are estimated from data via the evidence-maximisation iterative procedure of Tipping (2001), matching sklearn exactly.

| Parameter | Default |
|---|---|
| `max_iter` | `300` |
| `tol` | `1e-3` |
| `alpha_1` | `1e-6` |
| `alpha_2` | `1e-6` |
| `lambda_1` | `1e-6` |
| `lambda_2` | `1e-6` |
| `alpha_init` | `nullopt` |
| `lambda_init` | `nullopt` |
| `compute_score` | `false` |
| `fit_intercept` | `true` |
| `copy_X` | `true` (no-op, deprecated annotation) |
| `verbose` | `false` |

**Fitted attributes:** `coef_`, `intercept_`, `alpha_`, `lambda_`, `sigma_`, `scores_` (when `compute_score=true`), `n_iter_`.

**`ARDRegression`** is identical in form but uses a per-feature `lambda_j` prior, achieving sparse Bayesian regression. Same parameter set with the addition of `threshold_lambda=1e4` for pruning features.

`predict(X, return_std=true)` returns `(MatrixType, VectorType)` — predictive mean and predictive standard deviation. The boolean is a template tag, not a runtime flag, so the return type is statically resolved.

### 4.8 manifold::{TSNE, Isomap, MDS, LocallyLinearEmbedding, SpectralEmbedding, UMAP}

A new `Manifold` module covering sklearn's manifold-learning surface.

| Class | Algorithm summary | sklearn defaults preserved |
|---|---|---|
| `TSNE` | Barnes-Hut t-SNE with KL gradient descent. | `n_components=2`, `perplexity=30.0`, `early_exaggeration=12.0`, `learning_rate="auto"`, `max_iter=1000`, `n_iter_without_progress=300`, `min_grad_norm=1e-7`, `metric="euclidean"`, `init="pca"`, `method="barnes_hut"`, `angle=0.5`. |
| `Isomap` | KNN graph → geodesic distances (Floyd-Warshall or Dijkstra) → MDS embedding. | `n_neighbors=5`, `n_components=2`, `eigen_solver="auto"`, `tol=0`, `max_iter=nullopt`, `path_method="auto"`, `neighbors_algorithm="auto"`, `metric="minkowski"`, `p=2`. |
| `MDS` | Metric and non-metric MDS via SMACOF. | `n_components=2`, `metric=true`, `n_init=4`, `max_iter=300`, `eps=1e-3`, `random_state=nullopt`, `dissimilarity="euclidean"`, `normalized_stress="auto"`. |
| `LocallyLinearEmbedding` | Standard, modified, hessian, and LTSA variants. | `n_neighbors=5`, `n_components=2`, `reg=1e-3`, `eigen_solver="auto"`, `tol=1e-6`, `max_iter=100`, `method="standard"`. |
| `SpectralEmbedding` | Laplacian eigenmaps. | `n_components=2`, `affinity="nearest_neighbors"`, `gamma=nullopt`, `n_neighbors=nullopt`, `eigen_solver=nullopt`, `eigen_tol="auto"`. |
| `UMAP` | Uniform Manifold Approximation and Projection. | `n_neighbors=15`, `n_components=2`, `metric="euclidean"`, `min_dist=0.1`, `spread=1.0`, `n_epochs=nullopt`, `learning_rate=1.0`, `init="spectral"`, `negative_sample_rate=5`, `random_state=nullopt`. |

**Licensing note (UMAP):** `umap-learn` is distributed under the BSD-3-Clause license, which is MIT-compatible. Skigen's UMAP implementation is an independent C++ port of the algorithm described in McInnes et al. (2018); attribution is included in the source header per BSD-3 norms.

**Eigenvalue work** (Isomap, LLE, SpectralEmbedding) reuses Eigen's `SelfAdjointEigenSolver` and `BDCSVD`; ARPACK-style truncated eigensolvers are provided via `Spectra` (header-only, MPL2) gated by `SKIGEN_ENABLE_SPECTRA` if the user opts in. The default falls back to dense decomposition for small problems.

**KNN graph construction** (Isomap, LLE, UMAP) reuses the existing `KNeighbors` machinery — no duplicate spatial-index code.

### 4.9 calibration::CalibratedClassifierCV

Probability calibration via Platt scaling (sigmoid) or isotonic regression, fit via cross-validated stacking.

| Parameter | Default |
|---|---|
| `estimator` | `nullopt` (sklearn 1.7 default = `LinearSVC()`) |
| `method` | `"sigmoid"` (`"isotonic"` alternative) |
| `cv` | `5` |
| `n_jobs` | `nullopt` (treated as `1`) |
| `ensemble` | `true` |

**Fitted attributes:** `classes_`, `calibrated_classifiers_`, `n_features_in_`.

**Implementation constraints:**

- Holds a vector of `(base_clone, calibrator)` pairs sized to `cv` folds when `ensemble=true`, or a single pair when `ensemble=false`.
- Sigmoid calibrator: 1-D logistic regression on the decision function output.
- Isotonic calibrator: implemented in `src/Isotonic/IsotonicRegression.h` (a new utility component covering sklearn's `IsotonicRegression` directly — also exposed as a public estimator).
- Compile-time tag dispatch on the base estimator's `decision_function` vs `predict_proba` (matching sklearn's `_get_response_method` logic).

### 4.10 isotonic::IsotonicRegression

Promoted to a public estimator since `CalibratedClassifierCV` depends on it.

| Parameter | Default |
|---|---|
| `y_min` | `nullopt` |
| `y_max` | `nullopt` |
| `increasing` | `true` (also accepts `"auto"`) |
| `out_of_bounds` | `"nan"` |

Pool-adjacent-violators algorithm (PAVA) implemented in O(n log n).

### 4.11 model_selection::{Grid,Randomized}SearchCV

Hyperparameter search over a `Pipeline` or any estimator satisfying `Fittable`.

```cpp
template <typename Estimator>
class GridSearchCV : public Predictor<GridSearchCV<Estimator>, typename Estimator::ScalarType> {
    GridSearchCV(Estimator base,
                 ParameterGrid grid,
                 int cv = 5,
                 ScoringFn scoring = default_scoring_for<Estimator>(),
                 int n_jobs = 1);

    // Fitted attributes
    Estimator best_estimator_;
    ParameterDict best_params_;
    Scalar best_score_;
    std::vector<CVResultRow> cv_results_;
};
```

**Design questions resolved:**

- `ParameterGrid` is a `std::map<std::string, ParameterValue>` with `ParameterValue = std::variant<int, double, std::string, bool>`. Type-erased on string keys; estimators register their parameter setters via a small `set_param(name, value)` reflection layer.
- `RandomizedSearchCV` adds `n_iter` and per-parameter `Distribution<T>` — uniform / loguniform / categorical.
- Both run cross-validation through the existing `cross_val_score` path, with `n_jobs` parallelising over the grid (not over folds).

This requires every estimator gain a minimal `set_param(name, value)` / `get_params()` surface — see §5 for the required v1.0.0 break.

## 5. Required v1.0.0 Refactors (breaking)

The release theme permits — and requires — the following breaking refactors. Each is tied to one of the four motivating buckets in §1.

| Refactor | Bucket | Impact |
|---|---|---|
| `coef_` becomes `MatrixType` for all regressors (was `VectorType` in single-target case). | Multi-target | Callers must use `coef_.col(0)` for single-target. Convenience accessor `coef_vec()` provided for legacy code paths during transition. |
| `Estimator::set_param(std::string_view, ParameterValue)` and `Estimator::get_params()` added to base class. | GridSearchCV | Every concrete estimator must register its parameters. Default reflection macro `SKIGEN_PARAMS(...)` provided to keep boilerplate minimal. |
| `LinearRegression::n_jobs`, `positive` parameters added (previously omitted in v1.0.0). | sklearn parity | Constructor signature changes — old positional `LinearRegression(false)` still valid via default arguments. |
| `Pipeline::Pipeline(steps...)` accepts a uniform `(name, estimator)` tuple form (was positional only). | sklearn parity, GridSearch | Existing positional construction preserved as overload. Parameter-grid syntax (`"step__param"`) requires the named form. |
| `MetricsResult` aggregation type returned by classification metrics that previously returned `Scalar` for multi-output (e.g., `f1_score(..., average="macro")` returns `Scalar`; `average=None` returns `VectorType`). | sklearn parity | Affects callers that bound the result to `Scalar` unconditionally. |
| `KMeans::cluster_centers_` row/column ordering normalised to sklearn (rows = clusters). | sklearn parity | Verify; revert if already conformant. |

A migration note will be added to `doc/website/docs/development/` enumerating the breaks and the smallest-diff upgrade for each.

## 8. Out of Scope for v1.1.0

Deferred to v1.2.0 or v2.0.0:

- HMM / time-series estimators.
- Gaussian Processes (`GaussianProcessRegressor`, `GaussianProcessClassifier`).
- `BIRCH`, `OPTICS`, `DBSCAN`, `HDBSCAN`, affinity propagation, mean shift, agglomerative clustering — full unsupervised cluster zoo.
- `IsolationForest`, `EllipticEnvelope`, full anomaly-detection module.
- Cross-decomposition (`PLSRegression`, `CCA`).
- Sparse `MLP`, sparse manifold learners.
- Sklearn's `Pipeline.feature_names_out_` string-dtype machinery (Skigen has no string-dtype features yet).
- ONNX / PMML export.
- Dataset loaders (`fetch_*`, `load_*`).

Reserved for a future v2.0.0 (major) release:

- SIMD-explicit kernels for hot paths (where compiler auto-vectorisation falls short).
- GPU offload via SYCL / CUDA / Metal.
- Any further breaking API changes that are not motivated by the v1.1.0 buckets above.
